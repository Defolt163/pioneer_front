# PIONEER Frontend

Клиентская часть web-приложения PIONEER — агрегатора услуг для транспортных средств (мойка, шиномонтаж). Реализована на Next.js 14 с App Router. Взаимодействует с PIONEER Backend через REST API.

Исходной правдой для этого репозитория является текущий код. Эта документация описывает фактическое состояние проекта, включая известные ограничения.

## Что есть в проекте

- email-аутентификация без паролей — единый flow для входа и регистрации;
- прогрессивная регистрация: email → определение типа → имя (если новый) → код → токены;
- выбор типа услуги: мойка (`wash`) или шиномонтаж (`tire`);
- список организаций с фильтрацией по типу услуги и поиском по названию;
- просмотр состава услуги с выбором позиций и подсчётом суммы и длительности;
- выбор дня и времени записи (сегодня / завтра);
- создание бронирования с автоматической привязкой автомобиля пользователя;
- личный кабинет: email, управление автомобилями, история записей;
- автоматический рефреш JWT access-токена через refresh;
- обработка краевых случаев: нет автомобиля, нет организаций, истекшая сессия.

## Структура проекта

```
app/
  select-role/        — выбор роли (клиент / организация-партнёр)
  login/              — вход и регистрация по email-коду (единая страница)
  register/           — регистрация (альтернативный маршрут)
  services/           — выбор типа услуги
  organizations/      — список организаций с поиском и фильтрацией по типу
  service-details/    — состав услуги, выбор позиций, итог
  booking-time/       — выбор дня и времени записи
  booking-confirm/    — подтверждение и отправка бронирования на бэкенд
  profile/            — личный кабинет, автомобили, история записей
  org/connect/        — форма подключения организации-партнёра
  admin/requests/     — просмотр заявок организаций (для администратора)
  privacy/            — политика конфиденциальности

api/
  client.js           — базовый HTTP клиент с JWT-авторизацией и авто-рефрешем

services/
  authService.js      — отправка кода и верификация email
  bookingService.js   — создание бронирований
  organizationService.js - заявки организаций-партнёров
  servicesService.js - услуги и слоты записи

hooks/
  useAuth.js          — хук авторизации, хранение пользователя и токенов в localStorage

components/ui/        — переиспользуемые UI-компоненты (TopBar, Footer, Button и др.)
lib/
  constants.js        — роли, маршруты
```

## Auth flow

Основной публичный flow:

1. `POST /api/users/auth/send-code/` с email
2. Бэкенд возвращает `auth_type`: `login`, `registration` или `complete_registration`
3. При `registration` / `complete_registration` — пользователь вводит имя
4. `POST /api/users/auth/verify-code/` с email, кодом и name (если регистрация)
5. Бэкенд возвращает `jwt.access`, `jwt.refresh`, `session.id` и данные пользователя
6. Токены сохраняются в `localStorage`; при истечении access автоматически обновляется через refresh

Особенности:

- тип flow определяется бэкендом по состоянию пользователя в БД;
- в dev-режиме (`DEBUG=True` на бэкенде) письмо не отправляется, код печатается в консоль Docker;
- проверка кода всегда принимает `4444` в dev-режиме.

## Локальный запуск

### Требования

- Node.js 18+
- npm
- запущенный PIONEER Backend на `http://localhost:8000`

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка окружения

Создать файл `.env.local` в корне проекта:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000/api
```

По умолчанию клиент смотрит на `http://localhost:8000/api`. Если бэкенд на другом адресе — изменить переменную.

### 3. Запуск в режиме разработки

```bash
npm run dev
```

Открыть в браузере: `http://localhost:3000`

### 4. Сборка для продакшна

```bash
npm run build
npm run start
```
